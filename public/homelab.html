My Homelab

Over the past few months, I have been developing a homelab to practice my programming, networking, and IT skills. I have throughly enjoyed the process of building up systems which I can use to make my life a little easier and a little more fun. Self-hosting has been my main focus with the homelab, but I do plan to get into more experimental endevours as I continue. The following is a list of what I am using in my setup.

Hardware

  Currently my setup consists of an Omada ER605 router, a Netgear GS108 gigabit switch, two raspberry pis, and a Cisco UCS C220 M3 running Proxmox. All of my services (other than my media backup, for now), is run on vms inside the C220. My vpn and pihole run on one, my webserver on another, and a third currently for a minecraft server. The server is currently having some drive issues, so I haven't yet set it up for media storage yet. Eventually I do want to run pretty much everything through it, including media backups for me and my family, possibly a music server so I don't need to rely on Spotify, and a dynamic minecraft server webapp idea I have (news on that to come).

VPN

  I originally used the built in OpenVPN configuration in my router, I have since switched to using Wireguard with the server running on my C220. The convenience of being able to access and modify my set up from anywhere is one of the biggest things I wanted when starting my homelab. As a student I am seldom actually at my apartment, so being able to do my work from anywhere is very useful.

Media Backup

  While I currently don't do much in terms of backup, I do have all of my schoolwork on my laptop automatically backup to an ssd connected to a raspberry pi every 10 minutes. While this isn't the most safe way to do it (I'd much rather have it go to a raid NAS, but haven't gotten that set up yet), it ensures that if my laptop does ever die, I won't lose anything important as long as my ssd hasn't died yet. In writing this I realize that I really should have something periodically checking to see if it had died yet, and will keep this line up as a shameful motivation to do that.

Pihole

  Originally run on one of my raspberry pis, I now run a pihole on one of the VMs on my server. While predominately for blocking trackers and ads, I have also used it to assign my devices local addresses that the Pihole can resolve as them for ease of access (rasp1.domain.com is a lot easier to remember than an IP address). On a side note, I do find it pretty entertaining to watch the traffic going through the pihole and blocking any traffic that appears to be tracking or advertisements. A pihole isn't perfect, some websites like youtube are getting smart, sending ad traffic and content through the same urls so that it can't be blocked, but it's pretty convenient.

Webserver

  To run this website, and possibly any others, I am currently using nginx for a reverse proxy, and then expressjs to serve the content of the website. This isn't my long term plan, but I started putting this website on applications so I needed something. Eventually, I plan to switch to a Rust-based webserver, as I generally prefer Rust over Javascript. The website is just vanilla html, css, and js, but eventually I also plan to switch to a framework (possibly React, but I haven't 100% decided yet).

Minecraft Servers

  I have a vanilla minecraft server running using PurpurMC. I use an SRV record so that I can use a port other than minecraft's default while still making sure anyone joining the server doesn't have to type out an IP or port number. Eventually I want to run multiple servers off my C220, with many worlds stored on the drives and trusted users (my friends) being able to start up whichever server they want whenever they want.
